# CollabMatch → QoderWork 功能移植 Roadmap

> 从 QoderWork 借鉴核心能力，融合 CollabMatch 独有的协作匹配基因，打造「AI 协作网络 + 技能市场」。

---

## 产品定位对比

| 维度 | QoderWork | CollabMatch |
|------|-----------|-------------|
| 核心 | 个人 AI 效率工具 | AI 协作者匹配网络 |
| 模式 | 我 → AI → 完成 | 我 → AI → 找到你 → 一起完成 |
| 平台 | 桌面应用（Electron） | **网页版（浏览器 + 微信小程序）** |
| 技能 | 本地执行 | 云端执行 + 协作分发 |
| 用户关系 | 单用户 | 多用户网络 |

---

## 功能移植清单（按阶段）

### 🟢 Phase 1：技能市场（1-2 周）

移植 QoderWork 的核心功能，最小改动，最大价值。

| QoderWork 功能 | CollabMatch 移植方案 | 前端 | 后端 | 难度 |
|----------------|---------------------|------|------|------|
| **技能卡片** | 右侧面板「⚡ 技能市场」Tab | 新建侧边栏页 | 技能 CRUD API | ⭐⭐ |
| **官方/社区技能** | 技能分为 `official` / `community` | 分类标签 | 字段区分 | ⭐ |
| **安装技能** | 点击安装 → 出现在 ⚡ 下拉面板 | 安装按钮 | `userSkills` 关联 | ⭐⭐ |
| **技能详情** | 点卡片弹窗：描述 + 示例 + 安装次数 | Modal | `GET /api/skills/:id` | ⭐⭐ |
| **一键执行** | 已安装技能在 ⚡ 面板一键调用 | 已有雏形 | 已有 `runSkill` | ⭐ |
| **搜索技能** | 顶部搜索框，按名称/标签过滤 | 搜索 UX | `?q=&tag=` | ⭐ |

**数据模型扩展现有 SKILLS：**

```js
新增字段：
{
  isInstallable: true,          // 是否显示在技能市场
  category: 'official',         // 'official' | 'community'
  tags: ['原型', '设计'],       // 搜索标签
  installs: 2456,               // 安装计数
  author: 'CollabAI官方',       // 作者
  version: '1.2.0',
  screenshots: ['url'],         // 示例截图（可选）
}
```

**API 新增：**
```
GET    /api/skills/market              → 技能市场列表
GET    /api/skills/market?q=&tag=      → 搜索
POST   /api/skills/market/:id/install  → 安装技能
DELETE /api/skills/market/:id/install  → 卸载
```

### 🟡 Phase 2：任务编排 / 工作流（2-3 周）

QoderWork 的串联技能执行，变成 CollabMatch 的「协作流程模板」。

| QoderWork 功能 | CollabMatch 移植方案 | 难度 |
|----------------|---------------------|------|
| **工作流图** | 可视化流程画板（可选，后期做） | ⭐⭐⭐⭐ |
| **触发方式** | 手动启动 / 定时 / 事件触发 | ⭐⭐ |
| **节点串行** | 预设模板："PRD → 匹配 → 组队 → 协作" | ⭐⭐ |
| **节点并行** | "并行匹配不同领域专家" | ⭐⭐ |

**不做可视化画板，做预设流程模板：**

```
模板 1：「从想法到团队」
  1. AI 梳理 PRD     → 生成需求文档
  2. AI 扩展需求     → 补充市场分析
  3. 智能匹配         → 找到 3-5 位协作者
  4. 定向邀请         → 自动生成邀请文案
  5. 创建群组         → 自动拉群

模板 2：「项目体检」
  1. 诊断需求         → 三维度评估
  2. 优化描述         → 吸引协作者
  3. 匹配更新         → 重新推荐

模板 3：「每周协作报告」
  1. 周报生成         → 群组讨论总结
  2. 匹配推荐         → 本周新出现的人才
  3. 推送通知         → 发送给所有成员
```

**数据模型：**

```js
Workflow {
  id, name, desc, icon,
  steps: [
    { order: 1, skillId, title, config: {} },
    { order: 2, skillId, title, config: {} },
  ],
  triggers: ['manual', 'schedule'],
  author, installs, category
}
```

### 🟡 Phase 3：专家套件 / 角色包（1-2 周）

QoderWork 的专家套件 → CollabMatch 的「角色工具包」。

| 角色 | 套件内容 |
|------|---------|
| **产品经理** | PRD 生成 + 竞品分析 + 需求拆分 + 协作匹配 |
| **设计师** | UI 原型生成 + 设计规范提取 + 找开发者匹配 |
| **全栈工程师** | 技术方案生成 + 周期估算 + 找设计师匹配 |
| **创业者** | 商业计划梳理 + 融资对接 + 找技术合伙人 |
| **内容创作者** | 选题策划 + 脚本生成 + 找剪辑/运营伙伴 |

**交互：** 用户首次登录，根据注册角色自动安装对应套件。

---

### 🟠 Phase 4：社区 + 自定义技能（3-4 周）

| QoderWork 功能 | CollabMatch 移植 |
|----------------|-----------------|
| **自定义技能** | 用户可创建技能（写 Prompt）→ 审核上线 |
| **代码/脚本** | 支持用户上传 Node.js/Python 函数（沙箱执行） |
| **技能依赖** | A 技能输出 → B 技能输入 |
| **版本管理** | 技能版本号，回滚 |
| **评价/评分** | 用户对技能打分、评论 |

**这是差异化点：** QoderWork 的技能是个人工具，CollabMatch 的技能天然带着「协作」属性。一个技能可能触发一次匹配。

## 技术架构影响

| 关注点 | 当前 | Phase 1-2 后 |
|--------|------|-------------|
| 技能存储 | 硬编码 JS 对象 | 数据库（MongoDB） |
| 技能执行 | 前端内存 | 后端服务器 |
| 技能分发 | 无 | 版本化 + 审核流程 |
| 安全性 | 无 | 沙箱执行（vm2 / Docker） |
