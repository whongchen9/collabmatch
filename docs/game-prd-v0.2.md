## 1. 产品目标

在现有"需求匹配"应用之上，加一层像素风 2D 游戏化外壳（Canvas 渲染），将表单/列表 UI 转化为一个可走动、可交互的古代驿站场景，降低使用门槛、提升留存，但**所有底层协作逻辑不变**。

## 2. 核心用户故事

| # | 故事 | 验收标准 |
|---|------|----------|
| US1 | 作为新用户，我进入驿站场景，看到一个像素角色（我），可以用键盘走动探索，感觉像在玩游戏而不是填表 | 角色可用 WASD/方向键移动，场景有边界碰撞，有简单动画帧 |

### 4.3 视觉风格

- **画风**：像素风（pixel art），参考 16-bit RPG（如早期最终幻想/轩辕剑）
- **色板**：暖色调（木色、纸色、灯笼红），营造温馨客栈氛围
- **tile 大小**：32×32px，场景分辨率 640×480 基础，自适应缩放（`imageRendering: pixelated`）
- **角色大小**：32×32px（填充 tile，含脚底阴影）

### 5.2 数据流示意 [UPDATED]

```
用户按键输入 → Canvas 渲染循环（engine.js gameLoop）
                    ↓ (按 E 靠近交互点)
            GameBus.emit() 发送事件
                    ↓
        index.html 监听器 → 设置 renderTarget='game-overlay'
                    ↓
        调用现有 JS 函数 (renderSquareToOverlay / renderProfileToOverlay ...)
                    ↓
              api-bridge.js → Server API
                    ↓
              返回数据 → 更新 #game-panel-content DOM → 关闭 overlay 回到 Canvas
```

## 7. i18n 多语言适配 [NEW]

### 7.1 语言架构

现有 i18n 系统支持三种文本模式，通过 `t(key)` 函数统一访问：

| 模式 | 触发条件 | 品牌名 | 示例 UI 文本 |
|------|----------|--------|-------------|
| 中文（zh） | 浏览器语言为中文 / localStorage `cm_lang='zh'` | 需求匹配 | "进入驿站"、"退出驿站" |
| 英文（en） | 浏览器语言非中文 / localStorage `cm_lang='en'` | 需求匹配 / Need Match | "Enter Inn"、"Exit Inn" |
| 江湖模式（jianghu） | 中英文切换 ≥ 6 次（彩蛋） | **江湖故人** | "客官请进"、"辞别" |

## 8. 入口 UX 规范 [NEW]

### 8.1 进入驿站

```
[普通模式] 页面右上角 → 点击 "进入驿站" 按钮
    │
    ├─ 未登录 → showToast('请先登录后再进入驿站', 'warning') → 不切换
    │
    └─ 已登录 → enterGameMode()
                  ├─ gameModeActive = true
                  ├─ 隐藏 #app (display: none)
                  ├─ 显示 #game-canvas (class: active)
                  ├─ 按钮文本切换为 "退出驿站"，绑定 exitGameMode
                  ├─ initAudio() 初始化 Web Audio 上下文
                  ├─ loadGameScripts() 按需加载 6 个游戏模块
                  └─ initGame('game-canvas') 启动游戏循环
```

### 8.2 退出驿站

```
[游戏模式] 点击 "退出驿站" 按钮
    │
    └─ exitGameMode()
         ├─ gameModeActive = false
         ├─ stopGameLoop() → cancelAnimationFrame + 清理状态
         ├─ 隐藏 #game-canvas + #game-overlay
         ├─ 清空 #game-panel-content
         ├─ 显示 #app (display: flex)
         └─ 按钮文本恢复为 "进入驿站"，绑定 enterGameMode
```

### 8.3 交互——空格/E 键 [NEW]

当前实现使用 **E 键**（`KeyE`）。推荐下一步改为**空格键**作为主交互键，E 保留兼容：

| 键位 | 作用 | 优先级 |
|------|------|--------|
| `Space` | 主交互键（推荐，后续默认） | 新增 |
| `KeyE` | 兼容交互键（当前实现） | 保留 |
| `WASD` / 方向键 | 角色移动 | 不变 |

> 设计理由：空格键对非游戏玩家更直观（"按空格继续"是通用交互模式），降低学习成本。

### 8.4 Tab 键目标切换 [NEW]

当玩家附近（2 tile 曼哈顿距离内）存在**多个**可交互对象时：

| 动作 | 效果 |
|------|------|
| 按 Tab | 在高亮目标之间循环切换（NPC → 交互物 → NPC …） |
| HUD 提示 | 跟随高亮目标更新文本（如"按 E 对话 小二哥" → "按 E 查看 告示墙"） |
| 无高亮目标 | Tab 无效果 |

> 实现优先级：P1。小二哥附近通常不与其他交互物重叠，此功能主要在告示墙/铜镜双目标场景生效。

## 10. 技术风险 [UPDATED]

| 风险 | 影响 | 当前状态 | 缓解 |
|------|------|----------|------|
| 游戏循环与 DOM 渲染冲突 | 性能问题、卡顿 | ✅ 已缓解 | Canvas 和 DOM overlay 分层，弹窗打开时 `paused=true` 暂停游戏更新 |
| 单文件 HTML 过大 | 维护困难 | ✅ 已缓解 | 游戏模块拆到 `scripts/game/` 6 个独立文件，`loadGameScripts()` 按需串行加载 |
| 移动端体验差 | 触屏交互不合适 | ⚠️ 仍存在 | P0 不做移动端，桌面端优先 |
| ~~像素美术素材为零~~ | ~~视觉差~~ | ✅ 已消除 | 全部程序化绘制（OffscreenCanvas 逐帧生成），零外部素材 |
| RAF 内存泄漏 | 退出后帧循环不释放 | ✅ 已修复 | `stopGameLoop()` 调用 `cancelAnimationFrame` + `pause()` |
| i18n 硬编码 [NEW] | 游戏内 UI 文本多语言缺失 | ⚠️ 新识别 | "进入驿站"按钮、HUD 提示文本需接入 `t()` 函数 |
